# ADR-ML-001: 임베딩 모델로 OpenAI text-embedding-3-small

## 메타데이터
- **번호:** ADR-ML-001
- **날짜:** 2026-07-03
- **작성자:** 한재진 (코레일유통 IT)
- **레벨:** L1
- **상태:** 🔄 대체됨 (Superseded by ADR-ML-002)
- **관련 ADR:** ADR-ML-002(대체), ADR-SYS-002(추상화)
- **관련 프로젝트:** 규정 문서 RAG 아키텍처 비교

> ⚠️ 이 결정은 이후 ADR-ML-002로 대체되었으나, **당시의 판단 근거를 보존**하기 위해
> 삭제하지 않고 상태만 변경한다(조직 지식 보존 원칙).

## 1. 맥락 (Context)
초기 설계 시 한국어 규정 문서를 임베딩할 모델이 필요했다. 기술 스택 지정과
가용 키(OpenAI) 기준으로 임베딩 모델을 정해야 했다.

## 2. 결정 (Decision)
**채택(당시):** OpenAI `text-embedding-3-small` (1536차원). 스키마 `vector(1536)` 확정.

## 3. 대안 검토 (당시)
| 대안 | 장점 | 단점 | 판단 |
| --- | --- | --- | --- |
| text-embedding-3-large | 성능↑ | 비용·차원↑ | 실습엔 과함 |
| Gemini 임베딩 | 다국어 | 초기엔 키/검증 미확보 | 보류 |
| **text-embedding-3-small** | 저렴·검증됨·1536 | 한국어 법률문체 한계 가능 | **← 당시 채택** |

## 4. 근거 (Rationale)
1. 초기 가용 키가 OpenAI였고 검증 사례가 풍부
2. 1536차원은 pgvector HNSW에 적합, 비용 저렴(33k토큰 ≈ $0.0007로 전체 적재)
3. 실측 검색 품질 양호(유출 신고→제35조 1위 등)

## 5. 결과 및 트레이드오프
**긍정적 결과:** 94청크 적재·검색 정상, 저비용
**부정적 결과 / 위험:** 벤더 종속 + **크레딧 소진 리스크**(실제 발생 → 대체 계기)
**수용 기준:** 검색 재현율 확인 (충족했으나 운영 리스크로 대체)
