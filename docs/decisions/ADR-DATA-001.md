# ADR-DATA-001: 구조 기반 청킹 전략(장/절/조 + 별표 이중 인덱싱)

## 메타데이터
- **번호:** ADR-DATA-001
- **날짜:** 2026-07-03
- **작성자:** 한재진 (코레일유통 IT)
- **레벨:** L1
- **상태:** ✅ 승인
- **관련 ADR:** ADR-SYS-001(스키마)
- **관련 프로젝트:** 규정 문서 RAG 아키텍처 비교 (AI 챔피언 고급과정)

## 1. 맥락 (Context)
대상 문서는 개인정보보호 지침으로 장/절/조 계층 + 항/호/목 + 별표(표)가 혼재한다.
실사 결과 **마크다운 헤딩 표기가 불일치**(일부 조문만 `###`)했고, "제4장"이 두 번 등장하는
번호 중복, 표 셀 내 `<br>` 노이즈가 확인되었다. RAG 품질은 청크가 의미 단위를
보존하느냐에 크게 좌우된다.

**핵심 제약 조건:**
- 규정의 의미 단위는 "조(條)" — 조가 잘리면 답변이 불완전해짐
- 헤딩 기호를 신뢰할 수 없음 → 텍스트 패턴 인식 필요
- 표는 임베딩 시 마크다운 기호(`|`,`---`)가 노이즈

## 2. 결정 (Decision)
**채택:** 정규식 기반 구조 청킹. 규칙 R1~R9 요약:
- 1조 = 1청크(800토큰 초과 시 항/호 단위 분할 + 조 헤더 반복)
- 1별표 = 1청크
- 삭제 조문 포함(`content_type='deleted'`)
- 모든 청크에 `content`(원문)와 `embed_text`(브레드크럼+본문) **분리 저장**
- **표 이중 인덱싱**: 표는 규칙 기반 자연어 캡션으로 임베딩, 답변엔 원본 표 제시
- 장 등장 순번 `chapter_seq`를 유니크 키로("제4장" 중복 대응)

## 3. 대안 검토
| 대안 | 장점 | 단점 | 탈락 이유 |
| --- | --- | --- | --- |
| 고정 길이 청킹(500자) | 구현 단순, 범용 | 조문 경계 무시 | 조가 잘려 답변 불완전 |
| 마크다운 헤딩 파서 | 라이브러리 활용 | 헤딩 표기 의존 | 문서 헤딩 불일치로 오분할 |
| 표 원본 그대로 임베딩 | 단순 | 기호 노이즈 | 표 검색 품질 저하 |
| **정규식 구조 청킹 + 표 캡션** | 의미 단위 보존 | 문서별 패턴 튜닝 | **← 채택** |

## 4. 근거 (Rationale)
1. **실측 검증:** 94청크 생성, "보유기간 기준표" 질의에서 별표1이 유사도 0.729로 1위 → 표 이중 인덱싱 유효
2. **평가 재사용:** 골드 라벨을 `article_no`로 잡아 청킹 방식이 바뀌어도 QA 세트 재사용 가능
3. **결정성:** 규칙 기반 캡션은 무료·LLM 불필요 → 업로드 탭이 임베딩 키만으로 동작

## 5. 결과 및 트레이드오프
**긍정적 결과:** 조 단위 검색 정확, 표 질의 대응, 브레드크럼으로 짧은 조문 맥락 보강
**부정적 결과 / 위험:** 정규식이 이 문서 표기에 맞춰짐 → "제 1 조", "〔별표〕" 등 변형 표기
문서는 인식률 저하 가능(업로드 탭의 청킹 미리보기가 안전망)
**수용 기준:** 실문서 청킹 시 4개 content_type 모두 생성, 조번호 검색 재현율 확인
