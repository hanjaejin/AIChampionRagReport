# ADR-ML-002: 임베딩 모델 Gemini 전환 (ADR-ML-001 대체)

## 메타데이터
- **번호:** ADR-ML-002
- **날짜:** 2026-07-03
- **작성자:** 한재진 (코레일유통 IT)
- **레벨:** L1
- **상태:** ✅ 승인 (Supersedes ADR-ML-001)
- **관련 ADR:** ADR-ML-001(대체 대상), ADR-SYS-002(추상화)
- **관련 프로젝트:** 규정 문서 RAG 아키텍처 비교

## 1. 맥락 (Context)
Phase 7 전체 벤치마크 실행 중 OpenAI 임베딩이 `insufficient_quota`(429)를 반환했다.
쿼리 임베딩이 필요한 모든 검색 경로가 실패하고, 임베딩이 불필요한 조번호 직접조회만
동작했다. 챗은 이미 OpenRouter/Gemini를 쓰므로 임베딩만 OpenAI 비의존으로 전환해야 했다.

**핵심 제약 조건:**
- OpenAI 크레딧 없이 전체 벤치마크 실행 가능해야 함
- 기존 스키마 `vector(1536)` 유지(마이그레이션 최소화)
- **OpenRouter는 임베딩 API 미제공** → 임베딩 대안은 Gemini 또는 로컬

## 2. 결정 (Decision)
**채택:** Google `gemini-embedding-001`. 출력차원 1536(MRL)로 스키마 호환.
문서=RETRIEVAL_DOCUMENT / 질의=RETRIEVAL_QUERY **비대칭 task_type**. 레이트리밋
지수 백오프 재시도 내장. 전체 94청크 재임베딩. `OpenAIEmbeddingProvider`는 유지.

## 3. 대안 검토
| 대안 | 장점 | 단점 | 탈락 이유 |
| --- | --- | --- | --- |
| OpenAI 크레딧 충전 | 코드 무변경 | 벤더 종속 지속, 비용 | 근본 해결 아님 |
| 로컬 임베딩(BGE-M3 등) | 폐쇄망 정합·무료 | 로컬 인프라 필요, 실습 범위↑ | 학습 일정상 보류(운영 대안으로 기록) |
| OpenRouter 임베딩 | Provider 일원화 | **임베딩 API 없음** | 불가 |
| **Gemini gemini-embedding-001** | 1536 호환·다국어·키 보유 | 무료 티어 RPM 제한 | **← 채택** |

## 4. 근거 (Rationale)
1. **추상화 이점 실증:** `EmbeddingProvider` 교체만으로 파이프라인·테스트 무변경 전환(ADR-SYS-002)
2. **스키마 무변경:** output_dimensionality=1536로 기존 `vector(1536)` 그대로 사용
3. **품질 유지 실측:** 재임베딩 후 "보유기간 기준표"→별표1 1위, 3파이프라인 정상
4. **비대칭 임베딩:** 문서/질의 task_type 분리로 검색 적합도 향상

## 5. 결과 및 트레이드오프
**긍정적 결과:** OpenAI 비의존, 전체 벤치마크 실행 가능, 스키마 무변경
**부정적 결과 / 위험:** Gemini 무료 티어 RPM 제한(대량 배치 시 재시도 필요),
임베딩 토큰 사용량 미제공 → 비용 추정치 사용
**수용 기준:** 재임베딩 후 3파이프라인 end-to-end 정상 + embedding_version 갱신 확인(94/94)
