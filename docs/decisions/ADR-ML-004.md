# ADR-ML-004: 평가 프레임워크(심판 분리 + E1/E2/E3 오류분류)

## 메타데이터
- **번호:** ADR-ML-004
- **날짜:** 2026-07-03
- **작성자:** 한재진 (코레일유통 IT)
- **레벨:** L2
- **상태:** ✅ 승인
- **관련 ADR:** ADR-ML-003, ADR-DATA-001
- **관련 프로젝트:** 규정 문서 RAG 아키텍처 비교

## 1. 맥락 (Context)
3개 파이프라인을 정량 비교하려면 지표와 채점 체계가 필요하다. 검색 지표(Recall@k/MRR/nDCG)는
골드 라벨로 계산 가능하나, 답변 품질(faithfulness/correctness)은 LLM 심판이 필요하다.
또한 "오답이 검색·재정렬·생성 중 어디 책임인지"를 구분해야 개선 방향이 명확해진다.

**핵심 제약 조건:**
- 골드 라벨은 청킹 변경에 강건해야 함(조번호 기준)
- 심판이 자기 답을 채점하는 순환(self-judge) 회피
- 오류 원인의 자동 분류

## 2. 결정 (Decision)
**채택:**
- 지표: Recall@k·MRR·nDCG(순수 함수) + 비용·지연
- **LLM 심판을 생성 모델과 다른 모델로 분리**(예: 생성 gpt-4o-mini / 심판 gpt-4o)
- **오류 3단 분류**: E1(검색 한계: 골드가 후보군에도 없음) / E2(재정렬 실패: 후보엔 있으나
  최종 문맥 탈락) / E3(생성 실패: 문맥에 있는데 오답). E1/E2는 골드 라벨로 LLM 없이 자동 판정

## 3. 대안 검토
| 대안 | 장점 | 단점 | 탈락 이유 |
| --- | --- | --- | --- |
| 정확도 단일 지표 | 단순 | 원인 규명 불가 | 개선 방향 안 보임 |
| 심판=생성 동일 모델 | 비용↓ | 자기채점 편향 | 신뢰성 저하 |
| 사람 수기 채점 | 정확 | 확장성·비용 | 22×3 반복 부적합 |
| **다중 지표 + 심판 분리 + E1/E2/E3** | 원인 규명·자동화 | 구현 복잡 | **← 채택** |

## 4. 근거 (Rationale)
1. **원인 규명:** E1 비율이 3파이프라인 공통으로 높으면 임베딩 한계의 증거, E2는 rerank/라우팅 문제로 귀속
2. **자동 판정:** 골드=조번호라 E1/E2는 비용 0으로 판정
3. **실측:** 심판 분리 후 정상 판정(제36조 케이스 faithfulness/correctness=1.0/OK)

## 5. 결과 및 트레이드오프
**긍정적 결과:** 파이프라인별 오류 분포로 개선 방향 도출, 비용/정확도 동시 비교
**부정적 결과 / 위험:** 심판 LLM 비용·변동성, 절대 점수 임계치 튜닝 필요
**수용 기준:** 지표·비용·오류분류 단위 테스트 통과 + 실서비스 부분 검증(충족)
