# Phase 0 진행사항 요약 — 요구사항 재정의 및 리스크 진단

> 작성일: 2026-07-03

## 1. 수행 내용
- 워크스페이스 실사: `개인정보지침.md`(실습 대상 문서), `.env`(API 키), `plan.md`, `prompt/metaprompt.md` 확인. **git 저장소 아님, `.gitignore` 없음** 상태 확인.
- 실습 대상 문서(`개인정보지침.md`) 구조 분석:
  - 코레일유통 개인정보보호 지침 (전부개정 2024.06.27. 제2024-23호), 약 99KB / 1,290여 줄
  - 구조: 장(제1~4장) → 절(제2장 내 제1~7절) → 조(제1~72조+) → 항(①②)/호(1.2.)/목(가.나.) + 별표 1~14(마크다운 표)
- 클라우드 4종(Supabase, OpenAI, OpenRouter, Gemini, Cohere) 사용에 따른 보안 리스크 진단 수행
- 요구사항 모호 지점 11개를 질문(Q1~Q11)으로 정리하여 사용자 확인 요청

## 2. 주요 결정사항 (ADR 후보)
- **[확인됨] 청킹은 마크다운 헤딩이 아닌 정규식 기반 조문 패턴 파싱으로 해야 함**
  - 근거: 문서 내 헤딩 사용이 불일치함 (대부분 조문은 일반 문단, 일부만 `###` 헤딩. 장/절은 `###`, 별표는 `##`)
- **[제안] 실습 문서의 대외 공개 여부 확인 전까지는 회사 고유 정보(직책명 등) 치환본 사용 검토** — 사용자 확인 대기
- **[제안] 3개 파이프라인 비교 공정성 원칙: 동일 챗 모델·동일 컨텍스트 예산·동일 프롬프트 템플릿 고정** — Phase 1에서 확정

## 4. 이슈 및 해결
| 이슈 | 상태 |
| --- | --- |
| `.env`가 존재하나 `.gitignore`가 없고 git 저장소도 초기화 전 → API 키 유출 위험 | **미해결(긴급)**. git init 전 `.gitignore` 최우선 생성 필요. Phase 8까지 미루면 안 됨 |
| 문서 내 "제4장"이 두 번 등장 (개인정보파일의 관리 및 공개 / 보칙) — 장 번호 중복 | 인지됨. 청킹 메타데이터에서 장 번호를 유니크 키로 쓰면 안 됨 → Phase 1 설계에 반영 |
| 삭제된 조문 다수 존재 (예: "제16조 … 삭제") | 처리 방침 미정 (Q2로 사용자 확인 중) |
| 별표 표 셀 안에 `<br>` 태그 혼재 | 인지됨. Phase 2 청킹 구현 시 처리 |
