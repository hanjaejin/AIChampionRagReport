# Phase 2 진행사항 요약 — 청킹 모듈 코드 (TDD)

> 작성일: 2026-07-03

## 1. 수행 내용
TDD 사이클(Red→Green→Refactor)을 2회 수행했다.

**1차 사이클 (기본 구현):**
- 🔴 Red: Phase 1 설계 규칙 R1~R9를 14개 테스트로 작성 → `ModuleNotFoundError`로 전원 실패 확인
- 🟢 Green: `chunker.py` 구현 → 14개 전부 통과 (실제 `개인정보지침.md` 통합 테스트 포함)

**2차 사이클 (R2 확장 — 실측 데이터가 드러낸 개선):**
- 실제 문서 청킹 통계에서 `제3조(용어 정의)`가 약 1,042토큰인데 항(①)이 없고 호(1.)만 있어 분할 불가 확인
- 🔴 Red: "호 단위 분할 fallback" 테스트 추가 → 실패 확인
- 구조적 변경: 분할 로직을 `_segment_paragraphs`/`_group_segments`로 추출 (행위 불변 — 기존 14개 통과 유지 확인)
- 🟢 행위적 변경: 항 없으면 호(1. 2. …) 단위로 분할하는 fallback 추가 → **15개 전부 통과**

## 2. 주요 결정사항 (ADR 후보)
